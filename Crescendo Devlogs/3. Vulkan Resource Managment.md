At this point in the engine. I had decided that I wanted multi window capabilities. However my current implementation is not well suited to implementing this feature

Both [Vulkan Tutorial](https://vulkan-tutorial.com/) and [VkGuide](https://vkguide.dev/)Are good starting points for learning the basics of rendering in Vulkan. However both tutorials do not go into depth on the associations between each resource. Both tutorials use god-objects to represent the render state. While it provides a great overview and it is self contained, it does not lend itself well to useful features like multi-window rendering. Knowing the associations of resources with each other also allows us to create our own wrappers and encapsulate information within each other.
## C++ raii types exists??? Why not use those.
The key reason is because f**k you, I like reinventing the wheel, but also it would be incompatible with Vulkan bootstrap, there is an [issue](https://github.com/charles-lunarg/vk-bootstrap/issues/68) asking about support but it seems the repository authors are unwilling. Unfortunately, a limitation of the raii types is that they cannot take ownership of a raw handle, handles must be entirely created and destroyed by the raii type, hence I run my own raii types. This also allows me to better associate handles with each other saving a few bytes of memory
